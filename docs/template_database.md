# 🗄️ [템플릿 2] 데이터베이스(Supabase) 구축 및 테이블 스키마

유사 웹사이트 구축 시 Supabase 클라우드 데이터베이스에 생성해야 하는 테이블의 컬럼 구조와 RLS 정책, 초기 데이터 삽입용 SQL 스크립트입니다.

---

## 📊 1. 데이터 테이블 구조 명세

### 👥 1) `members` (회원 정보 테이블)
모든 모임 회원의 정보 및 직책을 담고 있는 테이블입니다.
* **`id`** : bigint, Primary Key (자동 증가)
* **`name`** : text (회원 성명, 필수)
* **`phone`** : text (연락처, 필수)
* **`company`** : text (소속/직장/직업)
* **`role`** : text (직책: 회장, 총무, 재무 등 지정 가능. 일반회원은 빈칸)
* **`photo`** : text (프로필 사진 이미지 Base64 코드 혹은 URL 링크)
* **`created_at`** : timestamp with time zone (생성일시)

### 📅 2) `schedules` (모임 일정 테이블)
다가올 모임의 일시, 회의 및 식사 장소를 관리하는 테이블입니다.
* **`id`** : bigint, Primary Key (자동 증가)
* **`title`** : text (모임 제목, 예: "2026년 신년 하례회")
* **`date`** : timestamp with time zone (모임 날짜 및 시간)
* **`location`** : text (2부 식사 장소명. 비워두면 자동으로 '추후 공지'로 매핑)
* **`created_at`** : timestamp with time zone (생성일시)

### 💳 3) `accounts` (공식 회비/상조 통장 테이블)
상단 위젯에 복사 버튼과 함께 렌더링될 공식 은행 계좌 정보를 가지는 테이블입니다.
* **`id`** : bigint, Primary Key (자동 증가)
* **`type`** : text (계좌 성격: `membership`(회비 통장), `mutual_aid`(상조 통장))
* **`bank_name`** : text (은행명, 예: "국민은행")
* **`account_number`** : text (계좌번호, 예: "000000-00-000000")
* **`owner`** : text (예금주명, 예: "남우회")
* **`created_at`** : timestamp with time zone (생성일시)

## 🛠️ 2. DB 초기 구축용 SQL 스크립트
Supabase의 **SQL Editor**에 들어가 아래 쿼리 전체를 붙여넣고 **`Run`**을 누르면 세 개의 테이블이 안전하게 한 번에 개설되며, 데모용 데이터가 삽입됩니다.

```sql
-- 1. members 테이블 생성
CREATE TABLE IF NOT EXISTS public.members (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name text NOT NULL,
    phone text NOT NULL,
    company text DEFAULT '',
    role text DEFAULT '',
    photo text DEFAULT '',
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 2. schedules 테이블 생성
CREATE TABLE IF NOT EXISTS public.schedules (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title text NOT NULL,
    date timestamp with time zone NOT NULL,
    location text DEFAULT '',
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 3. accounts 테이블 생성
CREATE TABLE IF NOT EXISTS public.accounts (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    type text NOT NULL UNIQUE,
    bank_name text NOT NULL,
    account_number text NOT NULL,
    owner text NOT NULL,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 4. 기본 계좌 더미 데이터 삽입
INSERT INTO public.accounts (type, bank_name, account_number, owner)
VALUES 
('membership', '국민은행', '000000-00-000000', '임시모임'),
('mutual_aid', '농협은행', '000-0000-0000-00', '임시상조')
ON CONFLICT (type) DO NOTHING;

-- 5. 보안 정책 RLS (Row Level Security) 비활성화
-- 소규모 친목 모임 앱 특성상, 토큰 관리 편의를 위해 익명 익스포트를 수용하도록 설정합니다.
ALTER TABLE public.members DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.schedules DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.accounts DISABLE ROW LEVEL SECURITY;
```
